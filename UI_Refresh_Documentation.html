Tooltip

Component HTML JavaScript Supporting Text Change Log

This is a basic tooltip. A basic tooltip is triggered when an form element is given focus.

This tooltip should only be used on text inputs, selects and textareas, not checkboxes and radios.

Your new password must contain

  • 8 to 12 characters,
  • a number,
  • a lower case letter,
  • an upper case letter, and
  • a special character from the following exclamation mark, dollar, percentage, caret, ampersand, asterisk, left bracket, right bracket, at sign, number sign, question mark, plus, hyphen, underscore ! $ % ^ & * [ ] @ # ? + - _
 

The password field is made up of two widgets:

  • A basic tooltip to display the password requirements information
  • A password check list widget which updates in response to user keystrokes to see if the password requirements have been met

Both widgets need to be initialised. See JavaScript tab for details.

Please retype your password instead of pasting.
 

The retype password field has a tooltip that is triggered by the paste event. In this example, the paste is prevented so that the user explicitly retypes their password to confirm their choice instead of copying from the password field, and the tooltip is display to inform them that they need to retype instead of pasting.

This is an example of an event triggered tooltip. Any event can be used to trigger and hide the tooltip, even custom ones.

Modal sizes

                        
Modals come in 3 sizes, small, medium and large:

                        
    
                            
  • The small modal has a class of small added to the interaction container
                                
     i.e. <div class="interaction cl-modal small"></div> 
    • 
                            
  • The medium modal has a class of medium added to the interaction container
                                
     i.e. <div class="interaction cl-modal medium"></div> 
    • 
                            
  • The large modal is the default modal so it does not require an additional class.
                                
     i.e. <div class="interaction cl-modal"></div> 
    •

Setting the active tab

                        
The active tab can be set by adding the class active
                            to the relevant anchor. This can be changed to a different class if necessary
                            - see activeClass in the options table below for details.

                        
When setHash in the options is enabled, the active tab will be
                            set by the hash in the url if this matches a panel id. If not, or if no hash is available,
                            the active tab defaults to the one set by the user using the active class.

                        
If no hash and no active class is set, the active tab will default to the first tab in the list.

                        
Options

                        
An object literal can be passed in as a second parameter into the constructor to override default behaviour and add additional functionality:


                        

                            
new ui.Tabs($('#tabsexample'), {
    activeClass: 'active',
    setHash: true,
    scrollToTabs: false,
    scrollPageToTabsTransition: "slow",
    tabAliasLinks: false,
    tabAliasLinksSelector: '.cl-tab-alias',
    dax: false,
    daxCallback: undefined
});
